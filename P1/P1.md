## 1. MUESTREO

En este apartado analizamos cómo la frecuencia a la que tomamos muestras define si somos capaces de reconstruir la señal original o si perdemos información por el fenómeno del aliasing.

### **a) Frecuencia de muestreo de la señal original**

Para obtener la frecuencia de muestreo (**fs_x**), calculamos el inverso del tiempo entre dos muestras consecutivas.

```matlab
Ts = t(2) - t(1); 
fs_x = 1/Ts; 
disp(fs_x); % El resultado es 160.000 Hz
```

### **b) Frecuencia de Nyquist (fs_min)**

La frecuencia de Nyquist representa la tasa mínima de muestreo necesaria para evitar el solapamiento espectral (aliasing). Matemáticamente es el doble de la frecuencia máxima (**fmax**) contenida en la señal.

```matlab
% Tras analizar el espectro de la señal facilitada x(t):
% Identificamos que el contenido frecuencial termina en 80.000 Hz aproximadamente.
fs_min = 160000; 
```

**Explicación para la defensa:** El Teorema de Nyquist-Shannon establece que para reconstruir una señal sin errores, la frecuencia de muestreo debe ser **mayor al doble de su componente de frecuencia más alta**. Si muestreamos por debajo, las réplicas del espectro se solapan y la señal se distorsiona.

### **c) Remuestreo de la señal (Señales g y h)**

Generamos dos nuevas señales cambiando su frecuencia de muestreo original mediante interpolación:

* **Señal g:** Muestreada a **fs_g = 1,5 * fs_min** (Cumple Nyquist, mucha resolución).
* **Señal h:** Muestreada a **fs_h = 2/3 * fs_min** (No cumple Nyquist, habrá aliasing).

```matlab
fs_g = (3/2) * fs_min;
fs_h = (2/3) * fs_min;

% Generamos los nuevos vectores de tiempo
t_g = (t(1) : 1/fs_g : t(end)).';
t_h = (t(1) : 1/fs_h : t(end)).';

% Interpolación spline para obtener los nuevos valores
g = interp1(t, x, t_g, 'spline');
h = interp1(t, x, t_h, 'spline');
```

### **d, e) Análisis espectral y conclusiones**

Al observar los espectros y las señales en el tiempo:

* **Aliasing:** En la señal **h(t)**, al muestrear por debajo de Nyquist, las frecuencias altas se "disfrazan" de bajas frecuencias. Esto deforma la señal original tanto en el dibujo temporal como en el espectro.
* **Energía de poca potencia:** El ruido que se ve en todas las frecuencias se debe a las **fugas espectrales** (por usar un tiempo de observación finito) y al error introducido por la propia interpolación **spline**.

## 2. CUANTIFICACIÓN UNIFORME

Aquí estudiamos la pérdida de precisión al redondear los valores de voltaje a niveles discretos (bits).

### **a, b, c) Parámetros del cuantificador de B bits**

Para un cuantificador con **B** bits totales (1 para el signo) y **D** bits dedicados a la parte decimal:

* **Niveles totales:** El sistema tiene **2 elevado a B** niveles de cuantificación.
* **Salto o Paso (Delta V):** Es la distancia mínima entre dos niveles, calculada como **2 elevado a -D**. Define la resolución del sistema.
* **Rango:** Define los valores máximo y mínimo que el sistema puede medir sin saturarse.

### **e) Ejemplo práctico: B = 5, D = 3**

* **Niveles:** 32 niveles distintos.
* **Paso (Delta V):** 0,125 Voltios.
* **Error máximo:** El error de redondeo nunca superará la mitad del paso, es decir, **+/- 0,0625 V**.

### **f, g, h) Cuantificación de k(t) y Error Cuadrático Medio (ECM)**

```matlab
% Ejemplo de cuantificación en MATLAB
q = quantizer('fixed', 'round', 'saturate', [5, 3]);
k_cuant = bin2num(q, num2bin(q, k));

% Cálculo del error cuadrático medio (ECM)
ECM = mean((k - k_cuant).^2);
```

**Explicación para la defensa:** Al aumentar el número de bits decimales (**D**), el escalón es más pequeño y el ruido de cuantificación disminuye (menos ECM). Sin embargo, si tenemos pocos bits para la parte entera, la señal se **satura** (se corta por arriba y por abajo), lo que dispara el error.

## 3. ANÁLISIS DE RESULTADOS (AUDIO)

Aplicamos todo lo anterior a un archivo de voz para entender el impacto auditivo de estos procesos.

### **a) Preparación del audio**

```matlab
[audio_x, fs_audio] = audioread('archivo_voz.wav');

% Eliminamos los silencios iniciales y finales (umbral 0.01V)
audio_util = audio_x(abs(audio_x) > 0.01);

% Cuantificamos a 7 bits (B=7)
% Si el audio está normalizado entre -1 y 1, usamos D=6 (1 bit signo, 6 decimales)
q_audio = quantizer('fixed', 'round', 'saturate', [7, 6]);
audio_7bits = bin2num(q_audio, num2bin(q_audio, audio_util));
```

### **b, c) Comparativa de calidad**

* **Auditivamente:** La señal de 7 bits presenta un ruido de fondo (siseo). Esto es el **ruido de cuantificación** producido por los saltos entre niveles.
* **Representación temporal:** Si hacemos zoom en la gráfica, la señal original es una curva suave, mientras que la de 7 bits tiene forma de **escalera**.
* **ECM del audio:** El error cuadrático medio nos permite cuantificar cuánta fidelidad hemos perdido respecto al audio original de alta resolución.

**Conclusión final para la defensa:** El proceso de digitalización siempre introduce errores. Un buen ingeniero debe elegir una frecuencia de muestreo que cumpla Nyquist (para evitar aliasing) y un número de bits suficiente para que el error de cuantificación sea imperceptible para el oído humano.
